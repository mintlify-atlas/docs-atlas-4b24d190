title: "ur_dashboard_msgs Package: Message and Service Types"
description: "Field-level reference for all types in ur_dashboard_msgs — RobotMode, SafetyMode, Load, SetMode action, and 28 service types used by the UR dashboard client."
---

The `ur_dashboard_msgs` package defines the ROS2 message and service types used to communicate with the UR Dashboard Server via the dashboard client node. All types are generated from IDL definitions at build time and are available under the `ur_dashboard_msgs` namespace. The package also includes one action type used by the robot state helper.

## Message types

<AccordionGroup>
  <Accordion title="RobotMode">
    Represents the current robot mode. The `mode` field holds one of the following integer constants:

    | Constant | Value | Meaning |
    |---|---|---|
    | `NO_CONTROLLER` | `-1` | No controller present |
    | `DISCONNECTED` | `0` | Driver is disconnected from the robot |
    | `CONFIRM_SAFETY` | `1` | Waiting for safety confirmation |
    | `BOOTING` | `2` | Robot is booting |
    | `POWER_OFF` | `3` | Motors are powered off |
    | `POWER_ON` | `4` | Motors are powered on but brakes are engaged |
    | `IDLE` | `5` | Brakes are released; robot is ready |
    | `BACKDRIVE` | `6` | Backdrive mode active |
    | `RUNNING` | `7` | Robot is running a program |
    | `UPDATING_FIRMWARE` | `8` | Firmware update in progress |

    ```
    int8 mode
    ```
  </Accordion>

  <Accordion title="SafetyMode">
    Represents the current safety mode of the robot. The `mode` field holds one of the following unsigned integer constants:

    | Constant | Value |
    |---|---|
    | `NORMAL` | `1` |
    | `REDUCED` | `2` |
    | `PROTECTIVE_STOP` | `3` |
    | `RECOVERY` | `4` |
    | `SAFEGUARD_STOP` | `5` |
    | `SYSTEM_EMERGENCY_STOP` | `6` |
    | `ROBOT_EMERGENCY_STOP` | `7` |
    | `VIOLATION` | `8` |
    | `FAULT` | `9` |
    | `VALIDATE_JOINT_ID` | `10` |
    | `UNDEFINED_SAFETY_MODE` | `11` |
    | `AUTOMATIC_MODE_SAFEGUARD_STOP` | `12` |
    | `SYSTEM_THREE_POSITION_ENABLING_STOP` | `13` |

    ```
    uint8 mode
    ```
  </Accordion>

  <Accordion title="SafetyStatus">
    A more detailed safety status type than `SafetyMode`, available on PolyScope 5. Includes additional states for ProfiSAFE and IMMI integrations. The `status` field holds one of the following signed integer constants:

    | Constant | Value |
    |---|---|
    | `NORMAL` | `1` |
    | `REDUCED` | `2` |
    | `PROTECTIVE_STOP` | `3` |
    | `RECOVERY` | `4` |
    | `SAFEGUARD_STOP` | `5` |
    | `SYSTEM_EMERGENCY_STOP` | `6` |
    | `ROBOT_EMERGENCY_STOP` | `7` |
    | `VIOLATION` | `8` |
    | `FAULT` | `9` |
    | `VALIDATE_JOINT_ID` | `10` |
    | `UNDEFINED_SAFETY_MODE` | `11` |
    | `AUTOMATIC_MODE_SAFEGUARD_STOP` | `12` |
    | `SYSTEM_THREE_POSITION_ENABLING_STOP` | `13` |
    | `TP_THREE_POSITION_ENABLING_STOP` | `14` |
    | `IMMI_EMERGENCY_STOP` | `15` |
    | `IMMI_SAFEGUARD_STOP` | `16` |
    | `PROFISAFE_WAITING_FOR_PARAMETERS` | `17` |
    | `PROFISAFE_AUTOMATIC_MODE_SAFEGUARD_STOP` | `18` |
    | `PROFISAFE_SAFEGUARD_STOP` | `19` |
    | `PROFISAFE_EMERGENCY_STOP` | `20` |
    | `SAFETY_API_SAFEGUARD_STOP` | `22` |

    ```
    int8 status
    ```
  </Accordion>

  <Accordion title="OperationalMode">
    Represents the operational mode of the robot. Uses string constants.

    | Constant | Value |
    |---|---|
    | `MANUAL` | `"MANUAL"` |
    | `AUTOMATIC` | `"AUTOMATIC"` |

    ```
    string mode
    ```
  </Accordion>

  <Accordion title="ProgramState">
    Represents the execution state of the currently loaded program. Uses string constants.

    | Constant | Value |
    |---|---|
    | `STOPPED` | `"STOPPED"` |
    | `PLAYING` | `"PLAYING"` |
    | `PAUSED` | `"PAUSED"` |

    ```
    string state
    ```
  </Accordion>

  <Accordion title="ProgramInformation">
    Holds metadata about a program stored on a PolyScope X robot. Returned in arrays by the `GetPrograms` service.

    | Constant | Value |
    |---|---|
    | `PROGRAM_STATE_DRAFT` | `"DRAFT"` |
    | `PROGRAM_STATE_FINAL` | `"FINAL"` |

    ```
    string name
    string description
    uint64 created_date
    uint64 last_saved_date
    uint64 last_modified_date
    string program_state
    ```
  </Accordion>

  <Accordion title="UserRole">
    Represents the current user role on the robot (CB3 only). Uses string constants.

    | Constant | Value |
    |---|---|
    | `PROGRAMMER` | `"PROGRAMMER"` |
    | `OPERATOR` | `"OPERATOR"` |
    | `NONE` | `"NONE"` |
    | `LOCKED` | `"LOCKED"` |
    | `RESTRICTED` | `"RESTRICTED"` |

    ```
    string role
    ```
  </Accordion>

  <Accordion title="VersionInformation">
    Holds a structured software version number. Returned by the `GetPolyScopeVersion` service.

    ```
    uint32 major
    uint32 minor
    uint32 bugfix
    uint32 build
    ```
  </Accordion>
</AccordionGroup>

    ```
    # Constants: controller, software, system
    ```

    ```
    bool success

    string report_id
    ```
  </Accordion>

  <Accordion title="GenerateSupportFile">

    ```

    ```

    **Response:**
    ```
    bool success
    string answer
    ```

## Action types

<AccordionGroup>
  <Accordion title="SetMode">
    Used by the `robot_state_helper` node to set the robot to a desired mode, optionally stopping and restarting the currently loaded program.

    **Goal:**
    ```
    # Target modes: 3 = POWER_OFF, 5 = IDLE, 7 = RUNNING
    int8 target_robot_mode
    bool stop_program    # Stop program before changing mode
    bool play_program    # Play program after target mode is reached (RUNNING only)
    ```

    **Result:**
    ```
    bool success
    string message
    ```

    **Feedback:**
    ```
    int8 current_robot_mode
    int8 current_safety_mode
    ```
  </Accordion>
</AccordionGroup>
